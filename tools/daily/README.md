# Daily scripture generator

Regenerates the daily "scripture of the day" assets for the 50 verse pool:
the two cards per verse, the landing pages, `src/data/daily.js`, and the SQL to
seed `comms.daily_pool`. Verse text is verbatim BSB, pulled from the study JSONs,
never typed by hand. No em or en dashes anywhere.

## Files
- `gen_cards.py`: the card renderer. Reproduces the fountain-ripple design in
  both formats (vertical 1080x1920 and landscape 1200x630) in the brand palette
  (Ink, Water, Light, cream) using Fraunces (display) and Newsreader (body).
  Verse size auto-fits so long verses stay clean.
- `match50.py`: matches each of the 50 references to its committed study JSON and
  pulls the exact BSB verse text, title, slug, and summary. Writes `match50.json`.
- `daily_build.py`: holds the 50 write-ups, renders all cards, builds every
  `public/daily/<slug>/index.html`, rewrites `src/data/daily.js`, and emits
  `pool_insert.sql` for the new `comms.daily_pool` rows.
- `match50.json`: the resolved reference to study join, regenerated by `match50.py`.

## Fonts (one time)
The renderer needs Fraunces and Newsreader as TTF. They ship as woff2 via
fontsource; convert once:

```
npm install @fontsource/fraunces @fontsource/newsreader
pip install fonttools brotli
python -c "from fontTools.ttLib import TTFont; \
import glob,os; [TTFont(p).save('/tmp/ttf/'+os.path.basename(p).replace('.woff2','.ttf')) \
for p in ['node_modules/@fontsource/fraunces/files/fraunces-latin-400-normal.woff2', \
'node_modules/@fontsource/fraunces/files/fraunces-latin-600-normal.woff2', \
'node_modules/@fontsource/newsreader/files/newsreader-latin-400-normal.woff2', \
'node_modules/@fontsource/newsreader/files/newsreader-latin-500-normal.woff2', \
'node_modules/@fontsource/newsreader/files/newsreader-latin-400-italic.woff2']]"
```

`gen_cards.py` reads the TTFs from `/tmp/ttf`. Adjust the `T` path at the top if
you vendor the fonts elsewhere.

## Run
```
python tools/daily/match50.py      # refresh match50.json from the studies
python tools/daily/daily_build.py  # render cards, build pages, write daily.js, emit pool_insert.sql
```

Then apply `pool_insert.sql` to Supabase (project oztgjzncxgobcszcwibp). The 5
originally seeded rows are left untouched by the guard delete plus insert.

## Adding or changing a verse
Add the reference to `match50.py` TARGETS and a write-up to `daily_build.py`
WRITEUPS (keyed by reference), then rerun both scripts. Any reference whose study
is not yet committed will fail the match, so only add verses whose study is live.
